# OSINT 商业情报与研报自动化生成引擎

> 输入一句话宏观指令，输出一份带引用来源的专业行业研报。

---

## 解决的核心痛点

传统商业分析师撰写一份行业研报，需要经历以下流程：

```
确定议题 → 拆解子问题 → 跨语种搜索 → 逐页阅读 → 数据清洗 → 交叉验证 → 逻辑推演 → 排版撰写
```

**其中 80% 的时间消耗在前三步：信息搜集与清洗。** 具体表现为：

| 痛点 | 表现 | 本系统的解法 |
|------|------|-------------|
| **跨语种检索低效** | 同一议题需要分别用中英文搜索、人工切换浏览器和翻译工具 | Planner 自动生成双语关键词，Researcher 并发执行中英文搜索 |
| **信息噪音大** | 搜索结果中混杂广告、软文、过时数据，人工过滤耗时 | 爬虫自动提取正文、去重、按来源类型分类 |
| **逻辑推演靠经验** | 从"A 事件"推演"B 趋势"依赖分析师个人经验，容易遗漏 | Synthesizer 基于多源数据进行结构化因果推演 |
| **缺乏质量校验** | 人工撰写的报告可能存在论点无数据支撑、逻辑跳跃 | Critic 像主编一样逐段审查，打回补充检索 |
| **全流程不可复现** | 每次写报告都是从零开始，无法复用之前的搜索和分析 | LangGraph checkpoint 记录每步状态，支持中断恢复 |

**一句话总结：** 把分析师从"信息搬运工"解放为"决策者"，将数天的研报撰写流程压缩到分钟级。

## 核心逻辑流：五 Agent 长链协作

本系统不是简单的"搜索 + 总结"，而是一条包含**任务拆解、并发采集、深度推理、质量校验、专业润色**的完整长链推理流水线。

### 整体流程

```
用户输入宏观指令
       │
       ▼
 ┌─────────────┐
 │   Planner   │  将宏观议题拆解为 5-8 个子检索任务
 │  (任务拆解)  │  输出：中英文双语关键词 + 优先级
 └──────┬──────┘
        │
        ▼
 ┌─────────────┐
 │ Researcher  │  并发执行所有子任务的搜索
 │  (信息采集)  │  工具：SerpAPI + 网页爬虫 + 学术搜索
 └──────┬──────┘  输出：结构化的原始发现 + 来源列表
        │
        ▼
 ┌─────────────┐
 │ Synthesizer │  对原始发现进行交叉验证和因果推演
 │ (综合分析)   │  模型：Claude Sonnet（长上下文 + 强推理）
 └──────┬──────┘  输出：结构化洞察 + Markdown 草稿报告
        │
        ▼
 ┌─────────────┐
 │   Critic    │  像主编一样审查草稿
 │ (质量审查)   │  检查：论点是否有数据支撑？逻辑是否自洽？
 └──────┬──────┘
        │
   ┌────┴────┐
   │         │
 通过      打回 ──→ 回到 Researcher 补充检索
   │
   ▼
 ┌─────────────┐
 │   Writer    │  润色为专业研报格式
 │ (报告撰写)   │  补充引用标注、数据来源附录
 └──────┬──────┘
        │
        ▼
   输出最终研报（Markdown，含引用来源）
```

### 关键设计决策

**1. 长链推理，而非单次调用**

区别于"搜索 → 丢给 LLM 总结"的简单模式，本系统将推理过程拆解为 5 个独立阶段，每个阶段有明确的输入/输出契约和独立的超时控制。这使得：

- 每一步可以使用最适合的模型（Planner 用低成本模型，Synthesizer 用强推理模型）
- 任意一步失败不影响已采集的数据，可以从断点恢复
- 中间结果可审计、可复现

**2. Critic 打回机制，而非一次性输出**

Synthesizer 生成的草稿会经过 Critic 审查。如果发现论点缺乏数据支撑或逻辑跳跃，Critic 会：

- 输出具体的审查意见（指出哪个段落、什么问题）
- 提供补充检索关键词
- 将任务打回 Researcher 重新采集数据

这模拟了真实编辑部"写稿 → 审稿 → 返修"的工作流，确保输出质量。

**3. 模型路由，而非统一模型**

不同 Agent 对模型的能力要求不同：

| Agent | 模型 | 原因 |
|-------|------|------|
| Planner | DeepSeek-V3 | 结构化输出、成本低 |
| Researcher | DeepSeek-V3 | 快速响应、并发调用多 |
| Synthesizer | Claude Sonnet | 长上下文理解、复杂推理 |
| Critic | DeepSeek-V3 | 规则性审查、成本低 |
| Writer | DeepSeek-V3 | 格式化润色、成本低 |

Synthesizer 是唯一使用 Claude 的环节，因为它需要同时处理 50+ 条原始发现并进行因果推演，这是长上下文 + 强推理的典型场景。

**4. 跨语种并发搜索**

Planner 为每个子任务生成中英文双语关键词，Researcher 并发执行搜索。这解决了单一语种搜索的信息盲区问题——很多行业数据只有英文报告才有，而政策和市场动态则以中文为主。

## 技术架构

```
┌─────────────────────────────────────────────────┐
│           Vue 3 前端 (Vite + Element Plus)       │
│     任务提交 · 实时进度 · 报告查看 · 历史列表      │
└──────────────────────┬──────────────────────────┘
                       │ REST API + SSE
┌──────────────────────▼──────────────────────────┐
│                  FastAPI 后端                     │
│       /api/tasks · /api/reports · /api/stream    │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│            LangGraph 编排层 (核心)                │
│   状态机驱动 · 条件分支 · 断点恢复 · 事件流        │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│                   工具层                          │
│   SerpAPI · 网页爬虫 · 学术搜索 · 社交媒体         │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│                   存储层                          │
│   SQLite (任务元数据) · 文件系统 (报告输出)         │
└─────────────────────────────────────────────────┘
```

**技术栈：** Python 3.11+ · LangGraph · LangChain · FastAPI · Vue 3 · Element Plus · SQLite · Docker Compose

## 快速开始

### 1. 配置环境变量

```bash
cd osint-engine
cp .env.example .env
```

编辑 `.env`，填入 API Key：

```env
DEEPSEEK_API_KEY=sk-xxx        # 必须，4 个 Agent 使用
ANTHROPIC_API_KEY=sk-ant-xxx   # 必须，Synthesizer 使用 Claude
SERPAPI_API_KEY=xxx            # 必须，网页搜索
```

### 2. 启动后端

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### 3. 启动前端

```bash
cd frontend
npm install
npm run dev
```

访问 http://localhost:3000

### 4. Docker 一键部署（可选）

```bash
docker-compose up --build
```
